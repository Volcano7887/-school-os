**Tech stack**: Next.js (App Router) + TypeScript + Tailwind + shadcn/ui, Supabase (Postgres + Auth + Storage). Deployed on Vercel at https://school-os-blue.vercel.app, repo at https://github.com/Volcano7887/-school-os.

**All core accounting modules are real and working**: Fees (collection, discounts/fines, search, due list, quick fees, carry-forward, types reference), Expenses, Salary, Cash Book, Ledger, Reports, Students, Classes, Analytics, Settings. Double-entry journal backs everything — `record_fee_payment`, `record_expense`, `record_salary_payment`, `record_cash_handover` all post atomically via SECURITY DEFINER Postgres functions with an audit log row in the same transaction.

**Recent work (this session, most recent first):**
1. **Responsive foundation pass** — replaced ad-hoc viewport breakpoints with a real system: `md`(768)=tablet start, `lg`(1024)=desktop start for top-level shell (sidebar/topbar), and `@container` queries for nested components whose real available width depends on a parent, not the viewport. Fixed a real page-overflow bug (topbar email had no max-width) and a real content-clipping bug (dashboard KPI cards got as little as 115px when a labeled sidebar left less room than assumed — fixed via a `@[1040px]` container-query threshold instead of a viewport breakpoint). Verified via direct DOM measurement (`scrollWidth`/`clientWidth`/`getBoundingClientRect`), not screenshots — screenshots previously produced a false "fixed" claim that turned out wrong.
2. **Fees Collection sidebar dropdown** — 6 new sub-pages (Search Payments, Search Due Fees, Quick Fees, Fees Type, Fees Discount, Fees Carry Forward) plus a real Carry Forward feature (`student_arrears` table) and Discount/Fine tracking on fee payments (Fine posts to a real "Fine Income" ledger account; Discount is a pure due-reduction, no cash movement).
3. **Cash Handover feature** — replaces the old disabled "End-of-Day Closing — Upcoming Feature" stub. Cash collected (ledger account 1000) accumulates with the accountant across days (no forced daily close); a Principal/Admin sees a live "Cash with Accountant" balance and a **Receive** button (role-gated — accountant can't self-confirm) that posts a real transfer journal entry to a new "Cash with Principal" account (1005) and logs the handover with who/when/note.
4. **Members management** (Settings → Members) — a school_admin can add anyone directly by email + role (Admin/Principal/Accountant): existing accounts get attached to the school, new emails get an account created on the spot via the Supabase service-role admin client, with a one-time temp password shown to share manually. Roles editable, members removable, all gated by an admin-only server-side role check on top of existing RLS.
5. **Indian-style fee receipt redesign + WhatsApp image sharing** — receipt page now has a real letterhead/ruled-table/amount-in-words (lakh/crore numbering)/signature-lines layout instead of a plain card. New `/api/receipts/[paymentId]/image` route renders the receipt as a PNG via Next.js's built-in `next/og` ImageResponse (no new dependency). `ReceiptShareButton` uses the Web Share API with files where supported (mobile — one tap into WhatsApp's share sheet with image + caption) and falls back to download-then-open-WhatsApp-text on desktop, since a plain `wa.me` link can't attach files without a paid WhatsApp Business API setup.

**Known infrastructure hiccup this session (resolved)**: GitHub had a platform-wide outage that silently broke Vercel's automatic-deploy webhook for several hours — 5 commits queued with zero deployment records despite multiple pushes and a forced empty-commit retrigger. Confirmed via a banner in the Vercel dashboard, not diagnosable from the GitHub API alone. Resolved on its own once GitHub's outage cleared; Vercel then worked through the backlog automatically. **Lesson: if deployments seem stuck for an unusually long time (much longer than the earlier-observed ~25min stalls), check the Vercel dashboard directly for a platform-status banner before assuming it's something in this repo.**
